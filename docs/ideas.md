# 💡 機能追加・更新候補アイデアリスト

Talklate4u の今後のアップデートで検討する機能のアイデア一覧です。

## 候補

### 1. グローバルショートカット（マイクミュート）

- **概要**: アプリがフォーカスされていなくても（ゲームプレイ中など）、ショートカットキーでマイクの「翻訳・認識」を一時停止/再開できる機能。
- **解決する課題**:
  - ゲーム中に咳払いや、配信に乗せたくない家族との会話が発生した際、裏画面に戻ってミュートするのが間に合わない。
  - OBS側でミュートしても、字幕生成アプリ側が動き続けていると翻訳が表示されてしまう。
- **実装イメージ**:
  - Electronの `globalShortcut` モジュールを使用。
  - 設定画面でキーバインドを変更可能にするのが望ましい（例: `Ctrl + F8`）。

### 2. Whisperの `initial_prompt` を用いたGlossary連動による認識精度向上

- **概要**: ユーザーがGlossary（用語集）に登録した専門用語を、Whisper APIの `initial_prompt` パラメータに動的に渡すことで、固有名詞などの音声認識精度を向上させる。
- **仕様・実装アプローチ**:
  1. **トークン数（文字数）の安全な足切り（クリップ）**:
     Whisperの224トークン制限を回避するため、内部ロジックで安全な文字数に切り捨てる。
     （例: 日本語・中国語なら上位から約100文字（約20語）、英語等のラテン文字なら約500文字で結合を打ち切る）
  2. **UIでの並び替え機能とガイドの提示**:
     Glossary設定画面に「⬆️⬇️矢印ボタン」等の並び順変更機能を追加する。あわせて「上位に配置した用語は、音声認識の精度向上にも優先的に利用されます」というTipを表示し、UXを向上させる。
  3. **Autoモード時の除外保護**:
     入力言語設定が「Auto（自動検出）」になっている場合にプロンプトを注入すると、Whisperの初期言語判定がプロンプトの言語に過剰に引っ張られ誤作動を起こす致命的なバグを避けるため、Auto時はプロンプトの注入を一切行わない（無視する）。
  4. **言語の適合と自然な結合**:
     現在の「入力言語」と一致するGlossaryの単語のみを抽出し、メタ的な指示文は含めず、純粋な単語のみを言語に合わせた自然な句読点（「、」や「, 」）で繋いで渡す構成とする。

### 3. プロファイル管理機能（用語集・設定のプリセット化）

- **概要**: 用語集（Glossary）、LLMの設定、VADの感度などを「プロファイル」として保存し、GUI上で瞬時に切り替えられるようにする。
- **解決する課題**:
  - ゲーム（Apex用、FFXIV用など）ごとに用語集を使い分けたいが、現状 `config.json` を直接編集したり別途管理したりするのは難易度が高い。
  - 配信内容（雑談・ゲーム・多言語コラボ）によって最適なVADの閾値や翻訳プロンプトが異なるが、その都度設定し直すのが面倒。
- **実装イメージ**:
  - 設定画面に「プロファイル保存・読込」機能を追加。
  - プロファイルごとに「用語集」「翻訳方向」「LLMモデル」「VAD設定」を紐付け。
  - 配信中にショートカットやボタン一つでプロファイルを切り替え可能にする。

### 4. マルチ言語の「スマート表示」モード

- **概要**: 2言語以上の翻訳結果を出す際、画面が字幕で埋まらないよう、コンパクトに表示または制御する機能。
- **解決する課題**:
  - 「マルチ言語表示をやりたいが、字幕が長すぎてゲーム画面が見えなくなる」という懸念。
- **具体的アプローチ（解決策）**:
  - **交互表示モード**: 1つの字幕エリアに、メイン言語とサブ言語を数秒おきにフェードで切り替えて表示。
  - **コンパクトモード**: メイン言語を大きく、サブ言語（例：英語）をその下に小さく控えめに表示。
  - **選択的表示**: 特定のキーワードが含まれる場合のみ、補助言語を表示する。

### 5. スマートTTS連携（発話重複防止機能付き）

- **概要**: 翻訳テキストを読み上げる際、配信者の生声やマイク入力と重ならないように制御する機能。
- **解決する課題**:
  - 「翻訳文をTTSが読み上げている間、配信者が喋り始めると音が被って邪魔になる」という懸念。
- **具体的アプローチ（解決策）**:
  - **ダッキング機能**: 配信者が喋り始めたら、TTSの音量を自動的に下げる（または瞬時に停止する）。
  - **キュー管理**: 配信者が喋っている間はTTSを待機させ、無音を検知してから読み上げを開始する。
- **実装イメージ**:
  - VOICEVOX やシステム TTS と連携。

### 6. リッチ・オーバーレイ・カスタマイザー（リアルタイムGUI編集）

- **概要**: 保存後の反映だけでなく、アプリ内のプレビュー画面でスライダーやカラーピッカーを使い、リアルタイムにオーバーレイのデザインを追い込める機能。
- **解決する課題**:
  - 現状でも保存すれば反映されるが、「もう少しフォントを大きく」「少しだけ透明度を上げたい」といった微調整を何度も設定画面とOBSを往復して行うのは非効率。
- **実装イメージ**:
  - フォントサイズ、色、縁取り（Stroke）、影（Shadow）、背景透過度、表示/非表示アニメーションの強さをGUIで調整。
  - 調整内容が即座にオーバーレイサーバー経由でOBS側に反映される。
